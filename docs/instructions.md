# スライド作成手順

## 手順1：事前確認

- `reference/seminar_slides.md`・本文・各章解説を参照し、対象章の改定内容を把握する
- 規準の文言・式・記号は**原典に忠実**に扱い、独自の言い換えや簡略化をしない
- 「変更なし」の項目は取り上げず、**変更点のみ**を抽出する
- 変更種別（追加・削除・移動・改定）を各項目に付記する

## 手順2：構成設計（トップダウン）

- 章全体の構成（スライド枚数・流れ）を先に決める
- 時間配分の目安：松尾（第1〜3章）約20分・ユーザー（第4〜6章）約20分・質疑約20分
- 1節＝1〜3枚を目安とし、**全体で60分以内**に収まる枚数を意識する
- 複雑な内容（数式・大きな表）は独立スライドに分離する
- 聴衆は構造設計者のため、**専門用語は省略せず正式名称で記載**する

## 手順3：Marpファイルの作成

ファイル先頭に以下のフロントマターを記述する（変更禁止）：

```yaml
---
marp: true
theme: default
size: 16:9
paginate: true
math: mathjax
style: |
  section {
    font-size: 20px;
  }
  h1 {
    font-size: 36px;
  }
  h2 {
    font-size: 27px;
  }
---
```

- スライド区切りは `---` を使用する
- 各スライドの先頭に `#`（章タイトル）または `##`（節タイトル）を置く

## 手順4：レイアウトルール（コンテンツ過密防止）

- 1枚あたりの項目数：**3〜5項目**を厳守
- 箇条書きの階層：**2段まで**（3段以上になる場合は別スライドへ分離）
- 表の列数：**最大5列**（超える場合は分割または縦持ち変換）
- 数式：**1スライド1式**を基本とし、導出過程は別スライドへ
- 文字数の目安：1項目あたり全角**40字以内**

## 手順5：表記ルール

- 節番号は規準体系に準拠（例：`## 602.3 ドリフトピン接合`）
- 変更種別を角括弧で付記：`[改定]`・`[追加]`・`[削除]`・`[移動]`
- 本文改定は `[本]`、解説改定は `[解]` で区別する
- 数式はLaTeX記法を使用する
  - インライン：`$...$`（例：`$E_{b0.05}$`）
  - 独立ブロック：`$$...$$`
  - コードブロック（` ``` `）内ではLaTeXを使用しない
  - `\text{}` 内の日本語は数式外に出す
- 略語は `docs/memory.md` の統一表に従う

## 手順6：Before/After比較の書き方

- 旧基準との対比は**必ず明示**し、表形式（旧・新・備考の3列）を優先する
- 表が使えない場合は以下のブロック引用形式を用いる
  ```
  > 旧：〜
  新：〜
  ```
- 数値・係数・式が変わった場合は旧値と新値を**並列で示す**（どちらが保守側かも補記）
- 変更の背景・理由が解説に記載されている場合は、1行以内で補足する

## 手順7：出力・納品

- ファイル名：`slides/slides_ch0X_YYYYMMDD.md`（X=章番号）
- 松尾担当分（第1〜3章）と自分担当分（第4〜6章）は**別ファイルで管理**する
- Marp CLIまたはVS Code拡張でPDFエクスポートできることを確認する
